# Credit Scoring

Решение классической задачи кредитного скоринга (прогнозирование вероятности дефолта клиента в течение двух лет) на основе популярного датасета **"Give Me Some Credit"**. 

В проекте реализован полный цикл разработки ML-решения: от глубокого исследовательского анализа данных (EDA) до построения оптимизированного ансамбля моделей машинного обучения с использованием автоматического подбора гиперпараметров.

## Стек технологий
* **Язык программирования:** Python 3.x
* **Анализ данных & Предобработка:** `pandas`, `numpy`, `scikit-learn` (StandardScaler, Pipeline)
* **Визуализация:** `seaborn`, `matplotlib`
* **Моделирование:** `scikit-learn` (Logistic Regression), `XGBoost`, `LightGBM`
* **Оптимизация гиперпараметров:** `Optuna`
* **Ансамблирование:** `scikit-learn` (VotingClassifier)
## Метрики финального ансамбля
После кросс-валидации и подбора гиперпараметров, финальный мягкий ансамбль (Soft Voting) показал следующие результаты на отложенной тестовой выборке:

* **ROC-AUC Score:** `85.60%` — показатель высокой разделяющей способности модели.
* **Accuracy:** `93.60%` — общая доля правильных ответов (с учетом сильного дисбаланса классов в датасете).
## Ключевые этапы реализации проекта

### 1. Предобработка данных & Инжиниринг признаков
* **Очистка данных:** Удаление дубликатов и неинформативных колонок (индексов).
* **Заполнение пропусков:** * Количество иждивенцев (`NumberOfDependents`) заполнено нулями.
  * Ежемесячный доход (`MonthlyIncome`) заполнен медианными значениями, вычисленными внутри пересечений возрастных групп (`AgeGroup`) и количества открытых кредитных линий/займов. Это позволило точечно восстановить пропуски без потери логики данных.
* **Обработка выбросов:** Отсечение аномально высоких доходов по квантилю `0.999` и удаление некорректных значений возраста.

### 2. Оптимизация гиперпараметров (Optuna)
Для каждой модели была написана универсальная функция валидации с использованием `StratifiedKFold` (на 3 фолда). С помощью фреймворка `Optuna` (по 20 итераций на модель) были подобраны оптимальные параметры:
* **Logistic Regression:** сила регуляризации `C`, тип штрафа (L1/L2) с оптимизатором `saga`.
* **XGBoost:** глубина деревьев, скорость обучения (`learning_rate`) и доля подвыборки (`subsample`) с использованием ранней остановки (`early_stopping_rounds=50`).
* **LightGBM:** аналогичная сетка параметров с оптимизированным процессом ранней остановки через коллбэки.

### 3. Построение финального ансамбля
Создан финальный конвейер (Pipeline) для логистической регрессии, включающий масштабирование признаков (`StandardScaler`), что предотвратило утечку данных (Data Leakage). 
Все три оптимизированные модели объединены в **VotingClassifier** с мягким голосованием (`voting='soft'`), что позволило усреднить предсказанные вероятности и снизить дисперсию ошибок индивидуальных алгоритмов.
